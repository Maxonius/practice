3. Исходные данные для создания программной системы:

3.1. Перечень решаемых задач:}

\renewcommand\labelenumi{\theenumi)}

\begin{enumerate}
\item исследовать особенности применения цифровых технологий в сфере туризма и выявить потребности пользователей в инструментах навигации и маршрутизации;
\item определить функциональные и технические требования к разрабатываемому веб-приложению, описать его назначение, структуру и условия использования;
\item разработать архитектуру программной системы, спроектировать интерфейс, структуру данных и взаимодействие с внешними сервисами;
\item реализовать программный продукт, обеспечивающий отображение туристических объектов, построение маршрутов и интерактивное взаимодействие с пользователем.
\end{enumerate}

{\parindent0pt
  3.2. Входные данные и требуемые результаты для программы:}

\begin{enumerate}
\item Входными данными для программной системы являются: координаты текущего местоположения пользователя;
список точек интереса из файла points.json;
параметры фильтрации (категория, тип маршрута и т.д.);
выбранные пользователем точки для маршрута;
отзывы и комментарии, вводимые через интерфейс;
действия пользователя в интерфейсе (нажатия кнопок, переключение меню);
данные геолокации и взаимодействие с картой через API Яндекс.Карт.
\item Выходными данными для программной системы являются: отображение отмеченных объектов на карте с балунами;
сформированный маршрут с указанием длины и времени;
визуализированная траектория пути с учётом выбранных точек;
список точек в маршруте и сохранённых в избранное;
отображение отфильтрованных объектов на карте;
подтверждение успешного добавления отзыва или маршрута;
графическое и текстовое отображение состояния интерфейса и карты.
\end{enumerate}
